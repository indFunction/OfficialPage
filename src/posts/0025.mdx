　Gitではフォークなどの機能を用いることで簡単にリポジトリーを複製できますが、そうではなく、完全に新たなリポジトリーとして従来のリポジトリーを引き継ぎたい、という場面が稀に存在すると思います。今回は、コミット履歴も含まれるリポジトリーを複製する方法を備忘録として纏めます。

<br />

1. GitHubで新しい（コピー先の）リポジトリーを作成します。

<br />

2. 適当なディレクトリーで、コピー元のリポジトリーのベアリポジトリー（Gitの更新情報だけを持つ／作業ディレクトリーを持たないリポジトリー）をクローンします。リポジトリー名の後に`.git`を付け忘れないようにしましょう。

<Code type='Bash'>
&#036; git clone --bare https://github.com/<ST type={['yellow_tc']}>&lt;ユーザー名&gt;</ST>/<ST type={['aqua_tc']}>&lt;コピー元のリポジトリー名&gt;</ST>.git
</Code>

<br />

3. クローンしたディレクトリーに移動し、コピー先にコピー元のリポジトリーをプッシュ（ミラープッシュ）します。

<Code type='Bash'>
&#036; cd <ST type={['aqua_tc']}>&lt;コピー元のリポジトリー名&gt;</ST>.git
&#036; git push --mirror https://github.com/<ST type={['yellow_tc']}>&lt;ユーザー名&gt;</ST>/<ST type={['lime_tc']}>&lt;コピー先のリポジトリー名&gt;</ST>.git
</Code>

<br />

4. 最後に、必要がなければクローンしたデータを削除します。

<Code type='Bash'>
&#036; cd ..
&#036; rm -rf <ST type={['aqua_tc']}>&lt;コピー元のリポジトリー名&gt;</ST>.git
</Code>

<br />

5. これでコピー先のリポジトリーにコピー元のリポジトリーが複製されました。お疲れ様でした。
